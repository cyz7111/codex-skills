# Hotfix 验证与回滚

## 必测路径

根据改动范围选择最小但有效的验证：

- 问题复现路径：修复前异常路径在修复后恢复。
- 旧业务路径：原本正常的核心路径仍正常。
- 接口状态：成功、失败、空数据、字段缺失、错误码。
- 页面状态：loading、empty、error、disabled、success、返回/刷新/重复提交。
- 平台差异：涉及多端时至少覆盖受影响平台。
- 环境差异：涉及环境配置时覆盖开发/测试/生产等配置来源。

## 命令验证

优先运行项目已有命令：

- lint
- typecheck
- unit test
- affected test
- build
- 小程序或平台构建检查

只写确认存在的命令。无法运行时说明原因和替代验证。

## 回滚方式

输出：

- 回滚文件。
- 回滚配置。
- 是否需要重新发版。
- 回滚后需要验证的最小路径。
