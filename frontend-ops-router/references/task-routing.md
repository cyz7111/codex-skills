# 前端任务分流参考

## 接手陌生项目

用于用户要求先熟悉项目、梳理模块、了解环境、接口、页面和业务配置的场景。

输出重点：

- 项目技术栈和运行平台。
- 启动、构建、发布和环境切换线索。
- 页面入口、路由、接口封装、状态管理和配置来源。
- 高风险公共点，例如请求封装、路由守卫、公共组件、全局 store、环境配置。
- 不直接改代码，除非用户明确进入实现阶段。

## 技术方案评审

用于技术方案、任务拆解、多人协作、排期和风险评估。

输出重点：

- 目标和非目标。
- 影响范围。
- 拆分任务。
- 依赖关系。
- 风险和回滚策略。
- 需要提前验证的技术假设。

## 工程化建设

用于 Monorepo、微前端、构建体系、CI/CD、组件库、规范、公共包和工具链治理。

输出重点：

- 先做现状调查，不直接替换工程体系。
- 分阶段方案：短期止血、中期收敛、长期治理。
- 迁移风险：构建、发布、环境变量、依赖版本、历史项目兼容。
- 验收标准：构建耗时、发版稳定性、复用率、回滚方式、文档更新。

## 平台差异

多平台任务必须注意运行平台差异：

- Web/H5：路由、浏览器兼容、响应式、缓存、鉴权跳转。
- 微信/支付宝/抖音小程序：平台 API、包体积、分包、登录授权、支付能力、审核限制。
- UniApp：条件编译、平台差异、运行端 API。
- 微应用：主子应用通信、样式隔离、路由同步、资源加载。
- 华为元服务：原生能力、页面配置、服务卡片或平台配置、构建发布链路。

平台不明时，将平台识别列为下一步待确认项。
