# Review Checklist

用于代码审查、风险检查和提交前质量评估。输出优先级是 bug 和风险，不是泛泛建议。

## 审查顺序

1. 真实功能 bug：错误条件、空值、竞态、状态不同步、路由参数、权限绕过。
2. 用户体验问题：无 loading/error/empty、重复提交、反馈不清、布局溢出。
3. 数据与接口风险：错误缓存、字段不兼容、异常被吞、绕过统一请求层。
4. 类型风险：错误断言、`any` 扩散、nullable 处理不完整。
5. 可访问性：键盘不可达、label 缺失、图标按钮无名称、弹窗 focus 问题。
6. 性能：不必要重渲染、大列表无分页、重复请求、昂贵计算。
7. 测试缺口：高风险逻辑无自动化或无手动验证路径。

## 输出格式

有问题时优先列 findings：

- 严重度：`P0` 阻断、`P1` 高风险、`P2` 中风险、`P3` 建议。
- 每条包含文件和行号。
- 说明触发条件、用户影响和建议修复方向。

没有问题时明确说未发现阻断问题，并说明仍未覆盖的验证范围。

## 注意

- 不因个人风格偏好提出低价值意见。
- 不要求重构无关代码。
- 对不确定问题标记为需要验证，不假装确定。
