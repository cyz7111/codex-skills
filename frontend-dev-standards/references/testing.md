# Testing

用于选择验证命令、补测试和提交前自检。

## 选择验证

优先运行能覆盖改动的最小命令：

- 类型变化：typecheck 或对应 package 的 `tsc --noEmit`。
- 组件逻辑：相关单测或组件测试。
- UI 流程：相关 e2e、story、浏览器手动验证。
- 构建配置或依赖变化：build。
- 样式局部变化：浏览器截图或人工视口检查。

只在项目文件或脚本明确存在时写精确命令。

## 测试重点

- 条件渲染：loading、empty、error、permission、disabled。
- 表单：校验、提交中、防重复提交、服务端错误。
- 列表：分页、筛选、排序、空结果、刷新。
- 数据流：缓存刷新、mutation 后状态、路由参数变化。
- 回归风险：共享组件、hooks、API client、全局样式。

## 没有测试时

如果项目缺少合适测试，不要硬造大型测试基础设施。优先：

1. 补一个贴近现有测试风格的小测试。
2. 或给出可复现的手动验证路径。
3. 明确说明未自动化覆盖的风险。
